Initial draft of Html & JavaScript for a tilde.town web ring.

~nick already has an implementation of a random link generator for tilde.town users. It is in vanilla JavaScript and certain to be more efficient and employ better practices than my own. You should probably use this instead.

My implementation generates a random link to a tilde.town user and a random link to another tilde box. You can see an example of its operation at the bottom of my home page. The code is available on github (and in my public_html directory):

  • the html
  • the javascript

If you want this on your page, it should suffice to drop the html wherever you'd like it. Forks and pulls are welcome, as is any manner of criticism on the code or approach. The thing relies on data generated by ~dan and ~pfhawkins. ~datagrock, vilmibm and others on irc were a tremendous help.

My bit of code generates a link to a tilde.town user and a random link to another tilde box. It is a flawed implementation and very bare-bones. Here is a list of its shortcomings:

  • It links jQuery. It links through the Google CDN so, I've been given to understand, it's probably already cached by browsers. But it should be reduced to vanilla JavaScript
  • It consists of two text links: no graphics no nothing.
  • It creates random links to tilde.town users. It's not ring-like since there's no opt-in and, as of now, only one other page has a similar gadget on their page, ~nick. (as ~dan pointed on irc).
  • I hope this will eventually be a universal html widget that can be dropped into a ~users page on any known box in the tildeverse, and it will generate links to their neighbors and to other boxes. It doesn't do this yet. ~dan and I have been talking.
